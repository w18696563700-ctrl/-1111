# 官方网站 V2 视觉升级施工说明

## 1. 总裁决

本轮 V2 视觉升级建议采用独立官网前端内的轻量 CSS 产品展示方案：不引入外部图片、不下载公网素材、不引入大型 UI 库、不改部署配置。

- 更稳：继续在 `apps/website` 内升级页面结构、样式 tokens 和展示组件。
- 更省成本：复用现有 Next.js / React / TypeScript / CSS 文件，不新增依赖。
- 更适合当前阶段：单页产品展示型官网，先把定位、首发能力、场景、边界和 CTA 表达清楚。
- 风险更大：为了视觉升级去改 Admin、Flutter、BFF、Server、Nginx、云端进程或直接使用未经确认授权的外部图片。

## 2. 当前最小闭环

官网 V2 仍围绕第一版最小闭环：

1. 展览项目展示。
2. 企业展示。
3. 项目沟通。
4. 资料协作 / 资料确认。
5. 消息楼。
6. 我的楼。
7. 首发三楼：展览、消息、我的。

本轮只升级官网前端视觉和首页信息结构，不改变 App、Admin、BFF、Server、合同、Nginx 或云端部署。

## 3. 需要保留但暂不开通

以下能力只作为边界说明或后续扩展位，不写成已开放能力：

- 装修楼。
- 全屋定制楼。
- 建材市场。
- 完整交易闭环。
- 支付收款、扣费、结算、退款、钱包、保证金、发票。
- 智能派单、AI 推荐、地图找厂、直播。
- 真实成交案例、客户评价、资质背书、数据看板。

## 4. 后续扩展位

后续可在不改变本轮边界的前提下预留：

- 替换为正式 App 首页截图或授权产品截图。
- 增加更多页面级 SEO 内容，但不做博客系统。
- 增加轻量图文资产，但必须使用自有或授权素材。
- 独立 Stage 进行云端发布和线上 smoke。

## 5. 设计来源与素材裁决

用户提供的 App 首页参考图给出本轮视觉方向：米白 / 白色背景、深绿色品牌主色、金色点缀、大圆角卡片、柔和阴影、强标题层级、产品截图主视觉和移动端 App 感。

仓库内检查到 `docs/04_frontend/screenshots/app_visual_system_v1_day7_after_exhibition_home.png`。该图是开发验证截图，包含桌面模拟器与外部界面，不适合作为官网公开产品图直接发布。

因此本轮采用 CSS 与结构化内容还原“App 首页视觉样机”：

- 不从公网下载图片。
- 不使用未经确认授权的展馆图片。
- 不编造真实项目、客户、成交、评价或资质。
- 在后续版本中等待用户补充可公开使用的正式 App 首页截图。

## 6. 页面结构

首页按以下顺序升级：

1. 顶部导航：Logo、首页、平台定位、核心场景、能力边界、适用对象、联系平台、右侧主 CTA。
2. Hero 首屏：平台归属、大标题、副标题、双 CTA、首发标签、右侧 App 首页视觉样机和轻量展馆空间背景。
3. 信任能力条：真实项目展示、资料协作共票、多角色协同、隐私与安全。
4. 首发三楼区：展览、消息、我的。
5. 核心场景区：项目展示、企业展示、项目沟通、资料协作。
6. 工作路径区：发现项目与企业、进入项目沟通、推进资料协作、回到我的楼。
7. 能力边界区：明确不承诺交易闭环、支付、AI、地图、直播、保留楼开放、虚构案例等。
8. 适用对象区：项目发布方、承接方 / 竞标方、展示企业、平台合作方。
9. CTA 区：先从一个展览项目场景开始沟通。
10. Footer：品牌说明、隐私政策、用户协议、联系平台。

## 7. 视觉 Tokens

本轮在 `apps/website/src/styles/tokens.css` 统一定义基础 tokens：

```css
--color-ink: #10251a;
--color-green-900: #0d3b2a;
--color-green-800: #124832;
--color-green-100: #e8f2ec;
--color-gold: #b8862b;
--color-gold-soft: #fff3d7;
--color-bg: #f7faf6;
--color-card: #ffffff;
--color-border: rgba(16, 37, 26, 0.12);
--color-muted: #66736c;
```

样式原则：

- 页面背景保持米白与白色。
- 主 CTA 使用深绿色。
- 高亮信息使用金色点缀。
- 卡片使用大圆角、浅边框和柔和阴影。
- 不使用花哨动效、复杂动画库或图表库。
- 移动端优先保证可读性，桌面端提升信息密度。

## 8. 文案边界

所有官网公开文案继续集中在 `apps/website/src/content/site.ts`。组件只负责渲染结构，不散落中文营销文案。

允许表达：

- 面向展览装修与展览定制场景。
- 支持项目展示、企业展示、项目沟通和资料协作。
- 支持消息与互动入口。
- 支持我的楼聚合个人、组织、认证、项目等入口。
- 当前首发围绕展览、消息、我的三楼展开。

禁止表达：

- 已实现完整交易闭环。
- 已实现支付收款、扣费、结算、退款、钱包、保证金或发票。
- 已实现合同签约、履约验收、智能派单、AI 推荐、地图找厂或直播。
- 装修楼、全屋定制楼、建材市场已公开开放。
- 大量真实客户案例、真实成交、生产全链路稳定运行。
- 内部 runtime、端口、进程、环境变量或支付通知 URL。

## 9. 验收标准

本轮 V2 视觉升级完成后需要满足：

1. 首页结构覆盖用户要求的 10 个区块。
2. Hero 右侧从抽象手机框架升级为产品化 App 首页视觉样机。
3. 不使用公网下载图片或未经授权图片。
4. 不引入大型依赖、CMS、动画库或图表库。
5. 公开文案集中在 `site.ts`。
6. `pnpm --filter website lint` 通过。
7. `pnpm --filter website typecheck` 通过。
8. `pnpm --filter website build` 通过。
9. 不修改 Admin、Flutter、BFF、Server、contracts、Nginx、env、lockfile 或 workspace 配置。
10. 不 SSH、不发布线上、不重启服务。

## 10. 本轮 No-Go 条件

出现以下情况必须停止：

- 视觉升级需要修改 Admin、Flutter、BFF、Server、contracts、Nginx 或 env。
- 需要新增大型 UI 库、CMS、动画库、图表库或数据库。
- 需要使用未经确认授权的公网图片。
- 需要把支付、交易闭环、虚构案例或线上稳定性写成官网承诺。
- 需要云端发布、Nginx reload、服务重启或 SSH。

当前未发现上述 No-Go。
