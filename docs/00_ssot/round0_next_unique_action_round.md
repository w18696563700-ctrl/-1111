# 《下一步唯一动作》

## 结论
- 本轮仍为 Round 0 盘点冻结，仅可进入**独立复核前提的资产盘点**。
- 当前不允许实施、联调、release。
- 下一步唯一动作必须是“派工文书收口”，不是实施口令。

## A. 下一步要先向谁发口令
1. 总控文书冻结：  
   - 派发对象：`总控文书冻结`  
   - 口令内容：提交当前工作流正式收口文书包，明确七角色现行链、历史六角色文书的沿用/降级关系，以及本轮标准模板，不得扩 scope。

2. 前端 Agent：  
   - 派发对象：`前端 Agent（仅本地）`  
   - 口令内容：提交本地前端盘点文书（现有页面、路由、消费端点、测试覆盖的盘点清单），不得提实现改动。

3. 后端 Agent：  
   - 派发对象：`后端 Agent（云端）`  
   - 口令内容：提交云端后端现状盘点文书（模块清单、关键 truth owner、现有迁移/状态机/审计证据），不得改代码。

4. BFF Agent：  
   - 派发对象：`BFF Agent（云端）`  
   - 口令内容：提交云端 BFF 现状盘点文书（路由聚合边界、当前 running package、server upstream 连接、错误码映射口径），不得改代码。

5. 结果校验 Agent：  
   - 派发对象：`结果校验 Agent`  
   - 口令内容：对本轮总控文书冻结文书、执行角色文书与本轮资产盘点结果做独立复核并给出 PASS / RISK / FAIL。

6. 联调发布 Agent：  
   - 派发对象：`联调发布 Agent`  
   - 口令内容：暂不下发布动作，仅待复核关闭后，再给出“可执行联调前准备条件清单（不执行）”。

## B. 各角色当前只能输出的内容
- 总控文书冻结：当前现行工作流文书包、引用链修订清单、沿用/降级说明；
- 前端 Agent：前端资产盘点（本地）与消费边界差异；
- 后端 Agent：云端后端真相盘点（状态机/路由/DB）；
- BFF Agent：云端 BFF 聚合边界盘点；
- 结果校验 Agent：独立签收；
- 联调发布 Agent：仅给出“前置证据缺口提醒”，不放行。

## C. 何时允许进入 Round 1
- 仅在以下条件同时满足后：
  1. Round 0 七角色所需文书齐全，其中 `总控文书冻结` 正式收口文书已回；
  2. 结果校验 Agent 对本轮复核项目给出明确签收；
  3. 核心阻断项仍在 `No-Go` 可追踪下，具备清晰关闭路径；
  4. `project_asset_inventory_round0` 与 `topology/tunnel` 快照与 `pre_unlock` 一致。

## D. 为何不是实现 / release
- 目前文书状态为 `No-Go for implementation / release`，且阻断项仍未闭环；
- 轮次要求为独立复核+盘点闭合，不是功能实现；
- 无法用“历史回执”替代可复核证据；不能把复核文本替换为发布结果。
