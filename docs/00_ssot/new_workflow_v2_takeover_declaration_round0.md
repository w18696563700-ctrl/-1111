# 《新工作流接管声明》

## 结论
- 本轮执行框架改为**新工作流 V2**：以“总控 + 总控文书冻结 + 5 执行角色”的七角色固定编制为唯一执行基准。
- 本轮为 **Round 0（现状盘点）**，是“盘点+冻结+派工前资产识别”，不授予实现许可。
- 当前阶段性裁决：**No-Go for implementation / release**，**Go for implementation 前独立复核**，阶段状态为 **phase0 安全边界**。

## A. 旧工作流处置
- 旧工作流中的“角色分工、派工关系、状态管理方式、回执流转方式”本轮全部废弃。
- 但旧工作流期内形成的历史资产（代码、页面、接口、数据库、配置、脚本、文档、发布记录）继续保留为有效项目资产，不得重复建设、不得平行重做、不得推倒重来。

## B. 新工作流硬性边界（V2）
固定角色为以下 7 名：
1. 总控（你）
2. 总控文书冻结
3. 前端 Agent（仅本地）
4. 后端 Agent（仅云端）
5. BFF Agent（仅云端）
6. 结果校验 Agent
7. 联调发布 Agent

除本轮新增并冻结的 `总控文书冻结` 外，不得再新增其他固定角色。

## C. Round 0 约束
- 本轮只允许产出盘点与复核文书，不允许：
  - 业务代码修改
  - 数据库变更
  - 本地 BFF/Server 代码编写
  - 实现/联调/发布动作
- 所有角色提交必须是“盘点口径”或“复核口径”，不得替代他人进入施工链路。

## D. 本轮立即生效边界
- 只允许 `/api/app/*`（App-facing）与 `/server/admin/*`（admin）体系；
- Flutter 仅通过 BFF 访问业务；
- BFF 只做聚合/转发/整形，不能持有业务真相；
- Server 为业务真相 owner；
- 文件上传遵循 `init -> direct upload -> confirm`，`objectKey` 不能作为业务真相。

## E. 本轮裁决语句
在 `pre_unlock` 与 `04_bff` 阶段门禁未闭环前，`实施放行/联调发布` 不得发起。
