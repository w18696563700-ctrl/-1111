# 我的楼专项主线 V1：本轮必做 / 本轮冻结占位 / 战略保留 三栏裁决表

> 本文件保留为历史 draft sidecar。
> 当前现行 canonical 文书改为 `docs/00_ssot/my_building_mainline_v1_three_column_ruling.md`。
> 本文件不再作为当前派工、门禁或主线裁决的优先引用依据。

## 1. 裁决规则

- `本轮必做` 只写当前主线已经成立、且本轮必须继续引用或继续推进的对象。
- `本轮冻结占位` 只写当前保留位置、但不进入重型实现的对象。
- `战略保留` 只写未来位，不得误写成当前已成立真相或当前放行动作。
- 本表只服务 `我的楼` 文书主线，不改写全仓 active implementation candidate inventory。

## 2. 三栏裁决表

| 本轮必做 | 本轮冻结占位 | 战略保留 |
|---|---|---|
| `我的楼` compact current-user hub 口径收紧 | `我的楼` 首层计数摘要只允许作为可选增强 | `profile` 下公共作者主页 |
| `我的楼 -> 我的项目` 私域入口与承接主线 | Flutter local `我的项目` route family 待在 `flutter_screen_map` 显式补挂 | `历史项目` finer buckets |
| Package 1 作为 account / organization / certification / device-security 引用底座 | `我的公司` richer edit / governance family 继续冻结占位 | full security center / governance center under `profile` |
| `building owner / route owner / page owner / truth owner` 分写 | `设置` richer groups 只保留 app-native slot | 第二工作台 dashboard |
| visible buildings 继续只写 `exhibition / messages / profile` | `项目工作台` 继续只保留摘要 / 导流定位 | hidden buildings runtime open-up |
| `docs-only freeze`、`implementation stage entry`、`implementation unlock` 三者分写 | `我的项目` 正式附件列表继续独立子线 | `profile` 承接 project / order / contract / fulfillment truth ownership |
| `我的论坛` 继续只保留 bounded first-level handoff | forum public author profile 只保留 owner boundary 与 future slot | 第二论坛首页 |

## 3. 三栏解释

### 3.1 本轮必做

- 这些对象当前已经成立，后续文书必须持续引用。
- 其中只有 `我的项目` 主线当前形成了进入 bounded implementation stage 的条件。
- 该结论不外溢到：
  - Package 1
  - 整个 `我的楼`

### 3.2 本轮冻结占位

- 这些对象当前可以保留名字、入口位、信息位或 route slot。
- 但当前不得把它们写成：
  - 已落成页面
  - 已开放 runtime truth
  - 已进入重型实现

### 3.3 战略保留

- 这些对象当前只能作为未来位记录。
- 当前引用它们时，必须同时写明：
  - 不是当前有效真相
  - 不是当前实现放行

## 4. 当前明确不代表的事项

- 本表不代表：
  - `我的楼` 已整体 implementation unlock
  - Package 1 已 runtime fully open
  - `我的公司` 已进入治理后台实现
  - Flutter local `我的项目` route family 已在 `flutter_screen_map` 正式补挂
  - strategic reserve 已自动进入当前阶段
  - 全仓 active implementation candidate 已切换为 `我的楼`

- 下一轮唯一动作：
  - 提交《我的楼专项 implementation 前阶段门禁核查表》

  - 判断 `我的项目` bounded implementation 是否具备文书前提
  - 维持 Package 1 `implementation No-Go`
- 该动作不得：
  - 直接派发 backend / BFF / Flutter / admin 代码实现
  - 直接宣布 `我的楼` 整体进入 implementation unlock
