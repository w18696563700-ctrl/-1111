# 《新工作流接管声明》

## A. 接管范围与效力

1. 本声明为 `Round 0` 文书，属于**盘点与冻结口径**，不授予实现许可。
2. 旧工作流在角色分工、派工关系、状态管理与回执流转上的执行方式，均视为**废弃**。
3. 旧工作流下已完成的资产（代码、页面、接口、数据库、配置、脚本、文档、发布动作记录）**继续保留为有效项目资产**，不得重复建设、不得平行重做、不得推倒重来。
4. 本项目继续使用“总控 + 总控文书冻结 + 5 执行角色”七角色新工作流，严格以现有 SSOT + 门禁规则为唯一执行基准。
5. 本轮目标仅为盘点，禁止直接编码、数据库迁移、部署发布。

## B. 新工作流硬性规则

- 固定角色：
  1. 总控
  2. 总控文书冻结
  3. 前端 Agent（仅本地）
  4. 后端 Agent（仅云端）
  5. BFF Agent（仅云端）
  6. 结果校验 Agent
  7. 联调发布 Agent
- 总控文书冻结负责 formal truth 收口、引用链修订、文书定稿与索引整理。
- 总控文书冻结不得替代总控做 Go / No-Go、scope 扩张或 implementation unlock 裁决。
- 阶段边界：`Round 0` = **现状盘点轮（No-Go for implementation / release）**。

## C. 本轮裁决边界

- 当前结论：`No-Go for implementation / release`。
- 当前允许：`Go for implementation 前独立复核`。
- 当前状态：`phase0 安全边界`。
- 任何后续动作必须先通过本轮盘点与独立复核。
