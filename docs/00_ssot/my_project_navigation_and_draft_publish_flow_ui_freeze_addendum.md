# 我的项目导航与草稿发布流展示层冻结 Addendum

状态：冻结

适用范围：本轮只覆盖 Flutter App 内“我的项目 / 我的项目详情 / 创建项目 / 编辑项目 / 草稿箱高亮落点”的展示层收敛。

## 1. 本轮最小闭环

本轮只做以下展示层修正：

1. “我的项目”“我的项目详情”“草稿箱高亮落点”“编辑项目”属于发布方私域工作台心智，页面路由可继续保留在 exhibition 路径下，但 bottom nav 选中态应显示“我的”。
2. 创建项目第一步提交前增加确认弹窗，提示本次只保存基础信息，不会正式发布，后续在我的项目草稿箱继续补资料、核对并完成发布。
3. 草稿卡片高亮文案不得使用“刚刚”这类非真实时间表达。有真实时间字段时展示“保存时间：YYYY-MM-DD HH:mm”；无真实时间字段时展示“已保存到草稿箱”。
4. 编辑项目草稿进入预发布列表的页面只保留一个清晰主动作：“确认保存到预发布列表”，次动作保留“仅保存草稿”。“当前生命周期”改为“当前任务 / 发布流程”口径，避免独立大卡重复占位。

## 2. 本轮不做事项

本轮不修改：

1. BFF。
2. Server。
3. OpenAPI / contracts。
4. 数据库 / migration。
5. 项目状态机。
6. 项目创建、保存、提交接口的真实动作。
7. bottom nav 三个主入口结构。
8. 云端服务配置、部署、重启或写入。

本轮禁止：

1. 伪造保存时间。
2. 伪造保存状态。
3. 新增接口字段。
4. 把前端展示兜底写成后端真值。
5. 把创建前确认弹窗解释为新的业务审核或新状态。

## 4. 字段真源表

| 展示项 | 允许真源 | 展示规则 | 无真源兜底 |
| --- | --- | --- | --- |
| 草稿保存时间 | 当前 `MyProjectListItemReadModel.publicProject` contract 未冻结 `savedAt`、`updatedAt`、`createdAt`；若 runtime payload 真实携带这些字段才可读取 | 按 `savedAt`、`updatedAt`、`createdAt` 优先级取第一个真实非空字段，格式化为 `保存时间：YYYY-MM-DD HH:mm` | `已保存到草稿箱` |
| 草稿高亮状态 | 当前路由 query `projectId` 与列表项目 `projectId` 匹配 | 只表示本次跳转落点，不代表后端新增状态 | 不展示高亮 |
| 创建项目确认 | 用户点击创建页提交按钮后的前端确认弹窗 | 确认后继续调用现有 `createProject` 动作 | 用户取消则不提交 |
| 保存到预发布列表 | 现有 submit/save action 返回的真实状态 | 只调整按钮文案和布局，不改变接口语义 | 失败按现有错误处理 |

## 5. 按钮能力表

| 按钮 | 本轮文案 | 真实能力 | 是否改接口 |
| --- | --- | --- | --- |
| 创建页提交 | 确认保存 | 弹窗确认后继续调用现有创建项目接口 | 否 |
| 创建页取消确认 | 返回修改 | 关闭弹窗，停留当前表单 | 否 |
| 草稿编辑主动作 | 确认保存到预发布列表 | 继续调用现有提交到预发布动作 | 否 |
| 草稿编辑次动作 | 仅保存草稿 | 继续调用现有保存草稿动作 | 否 |
| 我的项目卡片继续编辑 | 继续编辑 | 继续进入现有编辑项目路由 | 否 |

## 6. 验收标准

1. “我的项目 / 我的项目详情 / 编辑项目 / 草稿箱高亮落点”底部高亮为“我的”。
2. 创建项目点击提交前出现确认弹窗，确认后才调用现有创建动作。
3. 页面不再出现“刚刚保存到草稿箱”。
4. 有真实时间字段时展示“保存时间：YYYY-MM-DD HH:mm”；无真实时间时展示“已保存到草稿箱”。
5. 编辑项目页不再以“当前生命周期”作为独立大卡标题；口径统一为“当前任务 / 发布流程”。
6. 草稿进入预发布列表的主 CTA 唯一清晰，不出现上下重复的“确认保存到预发布列表”。
7. 不修改 BFF / Server / OpenAPI / 数据库 / 状态机。
8. 不新增 mock、不新增接口字段、不伪造时间或状态。

## 7. 风险与降级

1. 如果项目列表 payload 没有 `savedAt`、`updatedAt`、`createdAt`，只能展示“已保存到草稿箱”。
2. 如果 bottom nav 高亮与路由栈强绑定，本轮只做私域路由的最小 `currentBuilding` override，不重写 shell。
3. 如果现有测试依赖旧文案，需要同步更新测试断言，但不得扩大为业务逻辑重写。
4. 如果后续产品需要真实 `savedAt`，必须另开 BFF / Server 字段真源冻结，不在本轮前端伪造。
