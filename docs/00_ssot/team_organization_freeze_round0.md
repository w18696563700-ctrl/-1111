# 《团队组织冻结单》

## 目标
建立本项目固定协作编制，避免角色漂移与任务回流。

## 角色与职责

### 1. 总控
- 盘点现状、冻结边界、判定 stage/go/no-go；
- 下发派工，组织独立复核；
- 不直接替代执行角色编写代码。

### 2. 总控文书冻结
- 负责接管声明、组织冻结单、阶段门禁核查表、增量派工单、回执模板、引用链与索引收口；
- 只做 formal truth 整编与文书冻结；
- 不替代总控做主线裁决、Go/No-Go 或 implementation unlock。

### 3. 前端 Agent（仅本地）
- 页面、组件、路由、交互、消费层与本地测试；
- 通过隧道验证云端回归结果；
- 禁止在本地写 BFF / Server 代码。

### 4. 后端 Agent（仅云端）
- 云端业务模型、数据库、状态机、权限、审计、Admin 接口真相；
- 禁止本地写后端并宣称云端已完成。

### 5. BFF Agent（仅云端）
- 云端聚合接口、read-model 整形、错误归一、上下文转接；
- 禁止持有业务真相、禁止实现第二状态机。

### 6. 结果校验 Agent
- 做独立复核：重复建设、越权施工、漏做、伪完成、越界风险；
- 只输出复核判定，不代执行。

### 7. 联调发布 Agent
- 基于真实本地隧道与云端环境进行联调、回滚、发布前门禁检查；
- 禁止无复核放行。

## 禁止事项（逐角色）
- 总控：不得自行替代执行角色长期施工；
- 总控文书冻结：不得擅自扩 scope、不得独立发实施口令、不得改写总控裁决；
- 前端：不得本地发 BFF/Server 业务代码、不得临时规则硬编码；
- 后端/BFF：不得重复重建有效模块、不得绕开现有真实架构边界；
- 结果校验：不得只看回执做放行；
- 联调发布：不得无证据宣称可发布。

## 派工、回执与验收链
- 总控 → 先派工给总控文书冻结，收口当前工作流文书；
- 总控文书冻结 → 回执给总控；
- 总控 → 再派工给前端、后端、BFF、结果校验、联调发布；
- 前端/BFF/后端 → 回执给总控；
- 结果校验 → 回执给总控；
- 联调发布 → 回执给总控；
- 放行权仅在总控阶段裁决后触发。

## 阶段规则
- Round 0：仅可提交盘点与门禁复核文书；
- 下一阶段 `Round 1`：仅当本轮资产盘点 + 独立复核完成，且阻断项更新到位后才可考虑。
