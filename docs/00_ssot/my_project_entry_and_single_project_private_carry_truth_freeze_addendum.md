# 我的项目入口与单项目私域承接真源冻结单

## 1. Scope

- 本冻结单只覆盖 `我的项目入口与单项目私域承接 truth freeze`。
- 本冻结单只服务于：
  - `我的楼 -> 我的项目`
  - `我的项目` 列表
  - 单项目公域信息区
  - 单项目私域进度区
  - 项目正式完结 / 历史归档 / 待评价 / 已评价
- 本冻结单不进入：
  - contract freeze
  - persistence freeze
  - backend / BFF / Flutter 实现
- 本冻结单不扩到：
  - forum
  - 消息
  - 地图
  - 搜索界面
  - 地域分类页面
  - 企业库
  - 订单平台化后台
  - 合同后台
  - 履约治理后台
- 正式附件列表继续排除在本主线外。

## 2. Truth Freeze Conclusion

- `我的楼` 现正式新增 `我的项目` 作为私域项目资产入口。
- `项目工作台` 继续保留为摘要 / 导流页，不承担“我的全部项目”管理真相。
- `我的项目` 正式冻结为当前组织 scope 下项目资产的正式入口，不得与公域 `项目展示` 混同。
- `我的项目` 首页正式采用项目列表承接，首层分组正式冻结为：
  - `进行中`
  - `历史项目`
- 单项目页必须同时承接：
  - 公域信息区
  - 私域进度区
- `plannedEndAt` 只代表计划结束时间，不得直接推导为项目正式完结。
- `待评价` 是正式完结后的评价准入状态，`已评价` 只在评价动作完成后成立。
- 正式附件列表继续独立立项，不得混入本主线 truth。

## 3. 入口 Truth Freeze

### 3.1 `我的楼 -> 我的项目`

- `我的楼` 现正式新增：
  - `我的项目`
- `我的项目` 的正式真义是：
  - 当前组织范围内项目资产入口
  - 单项目继续处理入口
  - 私域项目管理入口
- `我的项目` 不是：
  - 公域项目入口
  - 公域项目展示入口
  - 仅最近项目快捷入口

### 3.2 `项目工作台` 与 `我的项目`

- `项目工作台` 继续保留为：
  - 总摘要页
  - 当前导流页
  - 最近项目上下文承接页
- `项目工作台` 当前允许继续承接：
  - `hasProjects`
  - `recentProjectId`
  - `recentProjectTitle`
  - `canCreateProject`
  - `canOpenProjectPool`
- `项目工作台` 现正式禁止被解释为：
  - 当前组织全部项目管理页
  - `我的项目` 的替代入口
  - 多项目列表真相 owner
- `我的项目` 与 `项目工作台` 现正式并存，且不得混同。

## 4. 归属 Truth Freeze

- `我的项目` 只看当前组织 scope 下的项目资产。
- 当前正式禁止：
  - 把公域 `project/showcase` 与 `我的项目` 合并为一个入口
  - 把 `recentProjectId` 继续当成“我的项目管理”主真相
  - 把 `项目工作台` 当前摘要 carrier 误写成项目资产主入口
- `我的项目` 的真义是：
  - 当前组织私域项目资产入口
  - 不是平台全局项目入口
  - 不是公域浏览入口

## 5. 列表 Truth Freeze

### 5.1 首页承接形态

- `我的项目` 首页现正式采用：
  - 项目列表
- 当前正式禁止：
  - 单实例承接
  - 仅最近项目承接
  - 用 `recentProjectId` 代表全部项目管理

### 5.2 首层分组

- `我的项目` 列表首层分组现正式冻结为：
  - `进行中`
  - `历史项目`
- 当前不直接冻结为：
  - `已完成`
- 原因已正式写死：
  - `已完成` 容易被误绑定到 `plannedEndAt`
  - 当前“项目正式完结”仍需独立业务真相
  - `历史项目` 更适合作为归档承接层

### 5.3 `历史项目` 的真义

- `历史项目` 现正式定义为：
  - 列表归档分层
  - 非当前活跃推进层
- `历史项目` 当前不自动等于：
  - 正式完结
  - 已评价
  - 争议 / 售后全部关闭

### 5.4 列表 item 的 truth 维度

- 列表 item 至少应承接两类 truth 维度：
  - 公域项目基础信息
  - 私域项目阶段摘要
- 当前正式允许：
  - 列表只展示最小公共信息与最小私域摘要
  - 完整私域继续处理内容留给单项目页

## 6. 单项目 Truth Freeze

### 6.1 双区结构

- 单项目页必须同时承接两大区域：
  - 公域信息区
  - 私域进度区
- 当前正式禁止：
  - 把公域信息与私域进度混成一个无边界大页

### 6.2 公域信息区

- 单项目公域信息区现正式复用已冻结的公域项目展示 truth，包括但不限于：
  - 标题
  - 建筑类型
  - 预算金额
  - 项目面积
  - 标准化地点
  - 地址与范围
  - 计划时间
  - 说明文案
- 公域信息区只承担：
  - 已发布项目公共真相的复用承接
- 公域信息区不承担：
  - 私域订单推进真相
  - 平台治理真相

### 6.3 私域进度区

- 单项目私域进度区现正式承接以下项目级私域 truth 维度：
  - 是否已接单
  - 当前订单状态
  - 合同状态
  - 履约进度
  - 验收状态
  - 争议 / 售后状态
  - 评价准入状态
- 私域进度区的正式真义是：
  - 项目级继续处理真相
  - 当前组织私域推进真相
- 私域进度区不是：
  - 公域展示真相
  - 平台治理后台真相

## 7. 状态 Truth Freeze

### 7.1 `plannedEndAt`

- `plannedEndAt` 现正式定义为：
  - 计划结束时间
- `plannedEndAt` 当前正式不得直接推导：
  - 项目正式完结
  - 历史归档
  - `待评价`
  - `已评价`

### 7.2 项目正式完结

- `项目正式完结` 现正式定义为：
  - 必须依赖独立业务真相
  - 不得由计划时间直接推导
- 当前正式禁止：
  - “计划结束即自动完成”

### 7.3 历史项目

- `历史项目` 现正式定义为：
  - 列表归档分层
  - 与是否评价完成不是同一真义
- 当前正式禁止：
  - “进入历史项目即已评价”

### 7.4 待评价

- `待评价` 现正式定义为：
  - 达到正式完结条件后
  - 进入评价准入状态
- `待评价` 不是：
  - 自动执行评价
  - 计划结束后的自动状态

### 7.5 已评价

- `已评价` 现正式定义为：
  - 评价动作完成后的状态
- 当前正式禁止：
  - “正式完结即自动评价”

## 8. 与现有主线的关系 Truth Freeze

- `我的项目` 不替代当前公域 `项目展示`。
- `我的项目` 不替代当前 `项目发布工作台`。
- `我的项目` 不替代当前 `项目工作台`。
- 三者关系现正式冻结为：
  - 公域展示
    - 看所有对外展示项目
  - 发布工作台
    - 创建项目
  - 项目工作台
    - 私域摘要与导流
  - 我的项目
    - 当前组织的项目资产与单项目承接入口
