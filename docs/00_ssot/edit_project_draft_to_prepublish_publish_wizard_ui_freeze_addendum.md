# 编辑项目草稿到预发布列表发布向导 UI 冻结 Addendum

状态：冻结

适用范围：本轮只覆盖 Flutter App 内 `编辑项目：草稿 -> 预发布列表` 页面的展示层精修。

## 1. 本轮最小闭环

本轮只做以下展示层优化：

1. 压缩编辑页顶部发布进度区，仍保留 `基础信息 / 报价依据资料 / 诚意金 / 确认发布 / 已发布` 五步。
2. 将“正在填写项目基础信息”从进度区内的大提示拆为轻量步骤提示卡。
3. 页面中部不再保留“当前任务”操作卡。
4. 草稿态的主操作、次操作和低频入口统一移动到底部固定操作区。
5. 当前内容核对、基础信息、项目地点与范围、补充说明、报价依据资料均保留，不减少字段和能力。

## 2. 本轮不做事项

本轮不修改：

1. BFF。
2. Server。
3. OpenAPI / contracts。
4. 数据库 / migration。
5. 项目状态机。
6. 保存草稿、保存到预发布列表、报价依据资料开放规则。
7. 上传三步流、附件绑定、正式发布规则。
8. bottom nav 三个主入口结构。
9. 云端配置、部署、重启或业务写入。

本轮禁止：

1. 新增假入口。
2. 新增假步骤。
3. 新增假发布能力。
4. 伪造报价依据资料可补充状态。
5. 删除已有字段或按钮能力。
6. 将共享发布进度组件直接改成全局紧凑样式，影响其他页面。

## 3. 页面结构冻结

| 区块 | 本轮规则 | 说明 |
| --- | --- | --- |
| AppBar | 保持现有编辑页标题和阶段标题 | 不改路由、不改状态。 |
| 发布进度 | 保留五步，草稿态高亮基础信息，编辑页使用 compact 模式 | compact 必须是可选能力，默认不影响其他页面。 |
| 当前步骤提示 | 轻量展示“正在填写项目基础信息” | 只提示下一步，不承接操作按钮。 |
| 当前任务操作 | 从页面中部移除 | 不减少能力，操作移动到底部固定区。 |
| 当前内容核对 | 保留卡片和展开 / 收起能力 | 默认逻辑不改，不新增审核状态。 |
| 基础信息 | 保留全部字段 | 只做视觉密度和分组优化。 |
| 项目地点与范围 | 保留省、市、区/县、详细地址、范围说明 | 不改联动逻辑。 |
| 补充说明 | 保持选填 | 不改变字段含义。 |
| 报价依据资料 | 草稿态展示轻量状态卡 | 不开放上传，不伪造可补资料状态。 |
| 底部固定操作区 | 草稿态固定展示主按钮、次按钮、低频入口 | 预留安全区，避免被 bottom nav 遮挡。 |

## 4. 按钮能力表

| 展示位置 | 按钮 / 入口 | 真实 action | 是否新增能力 |
| --- | --- | --- | --- |
| 底部固定区 | 确认保存到预发布列表 | 继续复用现有 `_submitProject` | 否 |
| 底部固定区 | 仅保存草稿 | 继续复用现有 `_saveProject` | 否 |
| 底部固定区 | 查看我的项目详情 | 继续复用现有 `_openMyProjectDetail(projectId)` | 否 |
| 当前内容核对卡 | 展开 / 收起当前内容核对 | 继续复用现有 `_toggleEditReviewExpanded(currentState)` | 否 |

## 5. 示意图修正规则

1. 视觉参考示意图，结构以本冻结单和口令为准。
2. 示意图中的中部“当前任务”大卡不直接落地。
3. 中部只保留轻量当前步骤提示和当前内容核对。
4. 底部次按钮必须是“仅保存草稿”，不是“返回草稿箱”。
5. 底部主按钮使用更严谨口径：“确认保存到预发布列表”。

## 6. 验收标准

1. 本轮只改 Flutter 展示层。
2. 不改 BFF / Server / OpenAPI / 数据库 / 状态机。
3. 不减少功能，不删除字段。
4. 页面中部不再出现“当前任务”操作卡。
5. 底部固定区展示“确认保存到预发布列表”“仅保存草稿”“查看我的项目详情”。
6. 报价依据资料草稿提示仍存在，但为轻量状态卡。
7. 发布进度 compact 只影响本编辑页，不影响我的项目详情等其他页面默认样式。
8. bottom nav 不遮挡底部固定操作区。

## 7. 风险与降级

1. 如果底部固定区与共享 `_SubmissionPageFrame` 冲突，优先在 `ProjectCreatePage` 局部用 overlay 实现，不改共享 frame。
2. 如果 Flutter test 仍被 native asset hook `Invalid SDK hash` 阻断，必须记录为环境阻塞，不写成测试通过。
3. 如果截图依赖重新启动 Flutter App 且本地构建阻塞，可先输出 No-Go 截图风险，不做云端或业务层绕行。
