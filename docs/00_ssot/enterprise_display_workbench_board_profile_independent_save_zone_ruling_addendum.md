# 《enterprise display workbench board profile independent save zone ruling》

## 1. 当前问题

当前 `优秀工厂工作台 -> 板块画像` 存在两个已证实的问题：

1. `工厂实景照片` 上传完成后会立即触发 `updateFactoryProfile(showcaseImageFileAssetIds)`，随后整页 `_loadWorkbench()` 回刷。
2. 回刷后的 `boardProfile` 会重新覆盖当前本地输入，导致用户尚未点击 `保存板块画像` 的文本/选项被冲掉。

这不是视觉层噪音问题，而是保存边界错误。

## 2. 方案裁决

### 2.1 不采用 dirty-field protection 作为主方案

`dirty-field protection` 只能在错误生命周期外侧补一层“防覆盖”。

它的缺点很明确：

- 仍然保留“图片上传 = 远端 profile write + 整页 reload”的错误链路
- 只是尽量避免覆盖，而不是从保存边界上消除覆盖
- 后续继续叠字段时，仍然容易出现新的漏保护点

因此，本轮正式裁决：

- `dirty-field protection` 不作为主方案
- 只允许作为 `local draft` 方案内部的最小辅助守卫存在

### 2.2 采用 local draft merge submit 作为唯一主方案

`板块画像` 正式收口为独立保存区。

对 `优秀工厂` 而言，当前区的保存规则冻结为：

1. 用户在当前区输入/选择的内容先进入本地草稿
2. `工厂实景照片` 上传成功后，只完成素材上传与本地草稿挂接
3. 在用户明确点击 `保存板块画像` 之前，不得因为图片上传而触发整块远端 profile write
4. 用户点击 `保存板块画像` 时，再把当前区的图片与文字字段一次性合并提交

### 2.3 当前区的主编辑内容

`优秀工厂 -> 板块画像` 的主编辑区固定为：

- 工厂名
- 工艺类型
- 核心产品
- 设备清单
- 厂房面积
- 工厂实景照片

这部分属于“当前展示主信息”，必须保持常显，并由同一个保存动作统一提交。

### 2.4 可选能力区折叠

以下字段正式降级为 `可选能力区`，默认折叠：

- 加急能力
- 加急周期说明
- 运输能力
- 支持仓储
- 月产能说明
- 最大订单承接能力
- 生产资质说明
- 配送半径说明

裁决要求：

1. 折叠区必须明确标记为 `选填`
2. 默认不干扰主编辑流
3. 不得与主编辑区共用“必填感知”

## 3. 前端实现边界

本轮前端允许做的事只有：

1. 重构 `板块画像` 为独立保存区
2. 将 `工厂实景照片` 从“上传即绑定远端 profile”改为“上传后进入本地草稿”
3. 将可选能力字段改为折叠区
4. 增加必要测试，证明：
   - 图片上传不会再导致未保存输入被整块覆盖
   - 可选能力区默认折叠

本轮不允许：

- 扩写 Server truth
- 新增第二套 profile save path
- 把其他区块一起改造成新的局部状态机
- 借机改动 submit / recommendation / public list/detail

## 4. 验收标准

本轮通过必须同时满足：

1. `工厂实景照片` 上传后，不再自动触发整块 `板块画像` 的远端回写与整页覆盖
2. 用户未点击 `保存板块画像` 前，本地区草稿不会因图片动作消失
3. `保存板块画像` 仍是当前区唯一正式提交动作
4. `可选能力区` 默认折叠，且主编辑区不受其干扰
