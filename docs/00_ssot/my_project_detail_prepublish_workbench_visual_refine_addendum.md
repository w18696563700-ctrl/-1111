# My Project Detail Prepublish Workbench Visual Refine Addendum

状态：本轮冻结

## 1. 本轮最小闭环

本轮只对 `我的项目详情（预发布补资料并发布页）` 做 Flutter 展示层视觉还原，把页面从普通纵向详情收敛为 `预发布补资料工作台`。

最小闭环：

- 项目基础信息卡从纯文字堆叠改成摘要工作台卡。
- 发布进度只展示 5 步状态，不展示长说明。
- 报价依据资料 checklist 保留五类资料，并强化图标、状态和当前选中态。
- 发布前待办在标准手机宽度下优先呈现 2x2 紧凑工作台；极窄屏才降级单列。
- 公共资源下载区默认折叠，数量从真实公共资源列表派生。

## 2. 当前差距冻结

| 模块 | 当前差距 | 本轮处理 |
| --- | --- | --- |
| 项目基础信息 | 仍接近文字堆叠，工作台感不足 | 改为摘要卡，核心字段优先，右侧轻量图标块 |
| 发布前待办 | 标准手机宽度仍是单列长列表 | 标准手机宽度优先 2x2，极窄屏单列 |
| checklist | 已有状态，但图标和主任务视觉不足 | 每项保留图标、状态、操作，当前项浅金底和金色描边 |
| 公共资源 | 已按真实列表派生数量 | 保持真实派生，不写死数量 |

## 3. 字段真源

| 展示项 | 真源 | 本轮规则 |
| --- | --- | --- |
| 项目名称 | `publicProject.title` | 缺失显示 `未提供` |
| 项目编号 | `publicProject.projectNo` | 缺失显示 `未提供` |
| 当前阶段 | `publicProject.state` 经前端 stage label 映射 | 不新增状态 |
| 待处理事项 | 阶段、报价依据资料、诚意金绿色通道组合派生 | 不造待办 |
| 报价依据资料状态 | 正式附件列表与当前待上传草稿附件 | 草稿附件只显示待上传，不算正式已补齐 |
| 发布前待办 | 三类必传附件、诚意金绿色通道、发布确认组合派生 | 未满足时不可提交 |
| 公共资源数量 | 公共资源真实列表按 `resourceCategory` 统计 | 显示 `合同模板 N`、`流程图与说明 N`、`公共资料 N` |

## 4. 组件边界

本轮允许局部调整：

- `my_project_detail_page.dart` 的项目摘要卡展示。
- `my_project_detail_prepublish_workbench_support.dart` 的发布前待办布局。
- `project_publish_progress_support.dart` 的进度紧凑参数。
- `project_attachment_panels.dart` / `project_attachment_widgets.dart` 的 checklist 展示层。

本轮不允许：

- 全局改附件上传三步流。
- 新增资料类型。
- 新增无真实路由的 `更换` 能力。
- 伪造公共资源数量或下载链接。
- 改 BFF、Server、OpenAPI、contracts、数据库、状态机。

## 5. 不做事项

- 不改 BFF。
- 不改 Server。
- 不改 OpenAPI / contracts。
- 不改数据库。
- 不改项目状态机。
- 不改诚意金规则。
- 不改上传三步流。
- 不新增假状态、假资料、假发布能力。
- 不删除上传、预览、打开、移除、刷新、发布、撤回、公共资源下载能力。

## 6. 验收标准

- 标准手机宽度下，发布前待办必须是 2x2 或明显双列工作台。
- 极窄屏才允许单列。
- 项目基础信息卡不再是纯文字堆叠。
- 报价依据资料是主任务区。
- 公共资源数量真实派生，不固定写死。
- 发布确认未满足时不可提交。
- bottom nav 不遮挡最后内容。
- `flutter analyze --no-pub` 无本轮新增问题。
- 相关 Flutter tests 通过，或明确记录既有环境阻塞。
