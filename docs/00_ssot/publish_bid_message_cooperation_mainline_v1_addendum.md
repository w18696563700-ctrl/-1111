# 发布-竞标-沟通-合作确认主链路冻结单 V1

## 1. 冻结结论

本文件冻结当前 App 普通展览项目的最小主链路：

`创建项目 -> 预发布补资料 -> 200 元项目真实性诚意金 -> 正式发布 -> 申请参与 -> 审核通过 -> 4000 元竞标服务费预授权额度 -> 提交竞标 -> 项目沟通 -> 发起合作确认 -> BidAward -> Order / Contract seed -> 后续成交确认`

本轮只允许 Flutter 展示层收敛，不修改 BFF、Server、contracts、OpenAPI、数据库、项目状态机、竞标规则、支付规则或消息楼业务真相。

## 2. 当前最小闭环

### 2.1 发布方链路

1. 发布方创建项目并保存基础信息。
2. 项目进入预发布核对，发布方补齐报价依据资料。
3. 正式发布前，发布方需完成 `200 元项目真实性诚意金`。
4. 支付是否完成、退款中、已退回等状态均以云端记录为准。
5. 项目正式发布后进入公域展示和竞标准入。
6. 发布方可在项目详情查看候选竞标方，沟通后发起合作确认。
7. 发起合作确认后，云端执行 `BidAward -> Order -> Contract seed`，但这不等于成交成立。

### 2.2 竞标方链路

1. 竞标方从展览页查看项目概要和项目详情。
2. 竞标方先申请参与，申请本身不等于提交竞标。
3. 发布方或平台审核通过后，竞标方可继续查看报价依据资料。
4. 提交正式竞标前，竞标方需完成 `4000 元竞标服务费预授权额度`。
5. 竞标方填写报价、上传附件并提交竞标。
6. 提交后通过项目沟通继续核对问题、接收结果通知和后续软链接。

### 2.3 平台闭环

1. Server 是项目、竞标、资金、BidAward、订单、合同锚点和成交确认的唯一业务真相 owner。
2. BFF 只做鉴权聚合、签名、整形、轻幂等、可见性裁剪，不拥有第二套业务真相。
3. Flutter 只做展示、用户输入、现有路由跳转和现有命令提交。
4. 消息楼只做项目沟通、状态通知、软链接承接，不做支付台、订单台、合同台、裁决台。

## 3. 术语冻结

| 场景 | 本轮统一口径 | 禁止口径 |
| --- | --- | --- |
| 发布方 200 元 | 项目真实性诚意金 | 押金、罚款、平台服务费、发单诚意金 |
| 竞标方 4000 元 | 竞标服务费预授权额度 | 已扣款、成交服务费、保证金 |
| 选择候选方 | 发起合作确认 / 确认合作并生成订单 | 轻量选择、立即成交 |
| BidAward | Bid 到 Order 的桥接真相 | 成交成立、合同已确认 |
| 消息楼 | 项目沟通 / 竞标沟通 / 结果通知 | 支付台、订单台、裁决台 |

## 4. 需要保留但暂不开通

1. 完整订单工作台、合同确认工作台、里程碑、验收、评价、售后。
2. 消息楼内直接支付、直接定标、直接成交确认。
3. BidAward 后的完整成交确认和扣费释放 UI。
4. 复杂候选方比较台、评分台、报价对比矩阵。
5. `converted_to_order` 后报价依据资料可见性的 Server/BFF 解锁修复。

## 5. 后续扩展位

1. 成交确认页：承接双方确认、成交金额、附件和最终服务费计算。
2. 消息楼确认卡：只承接软链接和状态摘要，不直接写业务状态。
3. 候选比较面板：在合同和候选资料真源冻结后再开。
4. 资金状态面板：只读聚合 200 元、4000 元、扣费、释放、退款状态。

## 6. 职责边界

| 层级 | 本轮职责 | 本轮禁止 |
| --- | --- | --- |
| Server | 只读核实业务真相 | 不改状态机、资金规则、BidAward、订单、合同 |
| BFF | 只读核实聚合和路由 | 不新增字段、不创造业务状态 |
| contracts / OpenAPI | 只读核实 | 不生成新合同、不改枚举 |
| Flutter | 展示层文案、信息层级、按钮语义收敛 | 不造假字段、不改路由真相、不新增 mock |
| messages | 沟通和软链接展示 | 不承接支付、订单、裁决、成交确认 |

## 7. 本轮验收标准

1. 旧口径 `P0-Pay`、`发单诚意金`、`询价报价单`、固定 `3%` 不出现在普通发布链路主体验。
2. `200 元` 在用户主体验中统一为 `项目真实性诚意金`。
3. `选择合作方` 语义调整为合作确认和订单/合同锚点生成，不误导为轻量选择或成交成立。
4. 消息楼文案统一为沟通、结果通知、软链接承接，不变成业务真相入口。
5. 不修改 BFF、Server、contracts、OpenAPI、数据库。
6. 不新增接口字段、不新增生产 mock、不改项目状态机。
7. `flutter analyze` 和相关 Flutter test 通过，或仅剩明确非本轮旧问题。
