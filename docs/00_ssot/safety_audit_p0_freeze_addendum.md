# Safety Audit P0 冻结单

## A. 冻结对象

本文件冻结 `Safety Audit P0` 横切底座边界。

本包只覆盖 P0 所需审计日志、内容快照、敏感词 / 保留词规则库，不进入 AI runtime、完整处罚体系或存量复扫。

## B. 本轮纳入项

| 能力编号 | 能力名称 | 当前冻结语义 |
| --- | --- | --- |
| CS-025 | 审计日志 | 对 P0 内容安全动作记录最小审计日志。 |
| CS-026 | 内容快照留存 | 对资料提交、举报目标、审核对象保留最小内容快照。 |
| CS-031 | 敏感词 / 保留词规则库 | 为 rule engine 提供 P0 最小敏感词与保留词规则库。 |

## C. 本轮明确不纳入项

- AI 审核服务统一接入层
- 存量内容复扫
- 用户违规累计分
- 完整处罚动作体系
- 完整申诉工单体系
- 多模型审核原始 payload 管理
- 风险词管理后台全功能

## D. 不纳入但保留在母版中的项

| 能力编号 | 能力名称 | 保留阶段 |
| --- | --- | --- |
| CS-027 | 处罚动作体系 | P1 |
| CS-028 | 申诉工单体系 | P1 |
| CS-032 | 用户违规累计分 | P1 |
| CS-033 | 存量内容复扫 | P2 |
| CS-034 | AI 审核服务统一接入层 | P1 |

## G. 当前禁止越界项

- 不得扩成完整申诉台。
- 不得启动存量复扫。
- 不得让 BFF 持有审计真相。

## I. 当前下游承接线程

后续承接顺序：

1. 后端线程承接审计日志、内容快照、规则库真相。
2. BFF 线程只承接必要错误和状态塑形。
3. 前端线程只消费受控提示与审核状态。
4. 结果校验线程独立复核留痕是否真实存在。

## J. 当前验收入口条件

进入 implementation 前必须满足：

- CS-025、CS-026、CS-031 已在追踪总表回写为已冻结。
- P0 runtime 已锁为 rule/manual。
- Profile Safety P0 状态机补充冻结已被引用。
- 总控复核允许进入 Safety Audit P0 implementation。

## K. 当前不允许进入实施的情形

任一条件成立则不得实施：

- AI 被写成 P0 runtime 前提。
- 审计日志被放到 BFF 当真相。
- 追踪总表未回写。
- 总控未输出子包冻结完成复核结论。
