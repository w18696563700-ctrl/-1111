# 旧项目名称申请历史只读收口冻结单

## 0. 总裁决

- 当前是否允许删除旧 `project_name_access` 历史数据：No-Go
- 当前是否允许删除旧 `project_name_access` route：No-Go
- 当前是否允许旧详情页继续作为审批入口：No-Go
- 当前是否允许把旧详情页降级为历史只读记录：Go

本包只处理旧「项目名称查看申请」详情页的历史只读收口。新的参与竞标申请仍是主流程，旧 route 仅作为历史兜底。

## 1. 本轮做什么

| 项 | 冻结结论 |
|---|---|
| Flutter 旧详情页 | 标题改为历史记录口径，说明旧能力已合并到申请参与竞标 |
| 旧审批动作 | 隐藏旧 `project_name_access.review` 主按钮和系统卡按钮 |
| 历史数据 | 保留展示状态、项目名称、申请记录和排障编号 |
| 新主链 | `bid_participation_request` 页面、审批、结果 CTA 不受影响 |

## 2. 本轮不做什么

- 不删除旧表、旧数据、旧接口。
- 不迁移历史 `project_name_access` 到 `bid_participation_request`。
- 不改 Server 状态机。
- 不改 BFF 业务真相。
- 不新增资料授权、NDA、水印或下载审计。
- 不触发竞标提交、预授权、支付初始化、回调或扣款。

## 3. 职责边界

| 层 | 边界 |
|---|---|
| Server | 保留旧历史读取能力和旧业务真相；本包不改状态机 |
| BFF | 继续转发旧详情读取；不新增状态判断 |
| Flutter | 只做历史只读展示和旧审批入口隐藏 |

## 4. 验收标准

| 验收项 | 标准 |
|---|---|
| 旧详情页标题 | 显示为历史项目名称查看申请 |
| 旧详情页说明 | 明确旧能力已合并到申请参与竞标 |
| 旧审批入口 | 不显示处理申请、同意查看项目名称、拒绝本次申请 |
| 新参与竞标页 | 继续显示处理申请、同意参与竞标等主链能力 |
| 历史兼容 | 旧 thread detail route 仍可打开 |
