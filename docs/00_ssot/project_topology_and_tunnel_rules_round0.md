# 《本项目拓扑与隧道使用规则》

## A. 拓扑边界（硬约束）
- 前端工程仅本地执行：`apps/mobile/**`
- BFF 与 Server 在云端开发/运行/部署；
- 本地开发不进行 BFF/Server 的代码编写。

## B. 固定隧道访问规则
- 统一隧道：
  - `ssh -N -L 8080:127.0.0.1:80 root@47.108.180.198`
- 本地访问映射口径：
  - 云端 80 端口映射到本机 `http://127.0.0.1:8080`
- 隧道用途：
  - 仅用于访问验证、联调前验证、证据补充；
  - 不等于远程编码通道；
  - 不等于云端直接开发替代方式。

## C. 角色知晓要求
- 必须明确知晓：总控、前端、结果校验、联调发布；
- 后端/BFF 明确知晓：其本地开发行为不替代云端，不以本地源代码为云端执行结果；
- 所有角色不得将“隧道可达”视为“release 成功”。

## D. 云端直接作业角色
- 后端 Agent（云端）
- BFF Agent（云端）
- 联调发布 Agent（云端验证/部署校验）

## E. 本轮禁止
- 在本地工程里以隧道名义直接“热更新/重建”云端模块；
- 在未核验隧道映射和 path 是否稳定前下发实现/联调发布口令。
