# 公共资料下载闭环与预发布按钮治理边界冻结单

状态：Go

## 1. 本轮目标

本轮在不修改 BFF、Server、数据库、状态机和公共资料真相的前提下，完成 Flutter 侧最小闭环：

- 公共资料下载从“打开浏览器链接”改为“App 内下载到本地后打开 / 分享 / 保存到文件”。
- 公共资料区清楚展示三类资料：合同模板、流程图与说明、公共资料。
- 区分“当前暂无该类资料”和“资料文件暂不可下载”。
- 预发布列表按钮去重。
- “作废删除”改为“作废并归档”，并解释不会硬删除。
- 已发布项目撤回到预发布前增加信用风险提示，但本轮不做真实扣分。

## 2. 当前最小闭环

只改 Flutter：

- 复用既有 `GET /api/app/project/public-resources` 读取公共资料目录。
- 复用既有 `GET /api/app/file/access?mode=download` 获取短期 `accessUrl`。
- 下载 bytes 后保存到 App 本地临时/应用目录。
- 下载完成后提供系统能力入口：打开、分享 / 保存到文件。
- 预发布按钮和撤回提示只做文案与交互治理，不改变后端动作。

## 3. 需要保留但暂不开通

- 信用分真实扣减。
- 信用分恢复、申诉、账本冲正。
- 公共资料后台配置、资源补齐和排序治理。
- 下载历史、离线资料库、断点续传。
- App 内 DOCX/PDF 解析预览器。

## 4. 后续扩展位

- 信用分账本：`credit_event` / `credit_adjustment` / `credit_appeal`。
- 公共资料下载审计和下载次数统计。
- 企业/项目维度资料包推荐。
- 文件预览缓存和离线管理。

## 5. 范围判断

| 范围 | 是否涉及 | 说明 |
| --- | --- | --- |
| Flutter | 涉及 | 本轮唯一实现范围。 |
| BFF | 不涉及 | 不新增 route，不改聚合，不改错误码真相。 |
| Server | 不涉及 | 不改公共资料、文件授权、项目状态机、信用分。 |
| contracts | 不涉及 | 上一轮已解锁 `public_resource` file/access，本轮只消费既有 contract。 |
| SSOT | 涉及 | 本冻结单与验收回执。 |
| 云端联调 | 只读涉及 | 只验证目录和 `file/access`。 |
| 数据库 | 不涉及 | 不建表，不写信用分，不补资料。 |

## 6. 云端只读核实

通过隧道 `http://127.0.0.1:8080` 验证：

- `GET /api/app/project/public-resources`：200。
- 返回 3 条资源：
  - `contract_template`：1 条。
  - `process_guide`：1 条。
  - `other_resource`：1 条。
- 三条资源均有 `fileAssetId`。
- 三条资源调用 `GET /api/app/file/access?mode=download&accessScope=public_resource` 均返回 200。
- 三条资源均返回 `accessUrl`。

## 7. 验收标准

- iPhone 点击公共资料下载不再直接跳浏览器。
- 下载完成后可通过系统面板保存到“文件”或分享。
- 三类公共资料都可见，并显示数量。
- 空分类显示“当前暂无该类资料”。
- 有资源但下载授权失败显示“资料文件暂不可下载”。
- 预发布列表不再出现同路由重复按钮。
- “作废删除”统一改为“作废并归档”。
- 作废确认明确进入已归档，不是硬删除。
- 已发布撤回到预发布前提示信用风险，但不写成已扣分。
- 无 BFF / Server / DB / 状态机改动。

## 8. 风险裁决

- 最稳：只改 Flutter 下载承接和文案，复用已通过云端验证的 `file/access`。
- 最省成本：不用新增下载接口，不接入重型下载管理器。
- 最适合当前阶段：先把用户能感知的下载闭环和按钮语义修顺，信用分进入下一轮。
- 风险最大：本轮直接实现信用扣分或绕过 `file/access` 拼 OSS 地址。
