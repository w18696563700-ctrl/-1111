# 消息楼交互枢纽与我的楼真实业务入口地图 V1

状态：Frozen

冻结日期：2026-05-07

冻结依据：用户已确认本文件口径，并允许进入 OpenAPI 裁决。

## 0. 总裁决

消息楼是 App 内交易与项目协作事件的交互枢纽，只承接提醒、入口、跳转、未读、上下文状态和轻量沟通，不承接资金、合同、支付、发票、钱包、保证金、最终金额、履约或争议的业务真值。

真实业务能力必须在 `我的楼` 或对应私域业务页面形成可独立完成的主路径。消息楼可以把用户从事件现场带到真实页面，但不能让聊天窗口成为第二业务中心。

本裁决不扩展支付、扣费、结算、钱包、发票、保证金、履约、验收、评价、争议或完整 IM。上述能力只能按后续专项逐一冻结。

## 1. Supersede 与不变范围

本文件只冻结“消息楼枢纽与真实业务页面入口归属”的平台级口径。

本文件不推翻：

1. 项目创建、草稿编辑、预发布补资料、发布确认主链路。
2. `draft -> submitted -> published` 状态链。
3. 上传三步流：init -> direct upload -> confirm。
4. `FileAsset` / `Evidence` / `ProjectAttachment` 真值归属。
5. `bid submit` gate 后移 Addendum。
6. BidAward 主路径。
7. Order / Contract seed 语义。
8. finalConfirmedAmount 主路径。
9. 200 元项目真实性诚意金既有冻结口径。

## 2. 平台入口总原则

| 原则 | 裁决 |
|---|---|
| 消息楼职责 | 展示提醒、未读、项目沟通入口、状态提示、routeTarget 跳转 |
| 我的楼职责 | 承接真实业务页面、历史记录、状态详情、用户可重复进入的主路径 |
| Server 职责 | 持有业务真值、状态机、审计、权限、资金/合同/确认真值 |
| BFF 职责 | 只做 App-facing 聚合、projection、错误归一、routeTarget 透出 |
| Flutter 职责 | 展示、路由、反馈、刷新、本地草稿，不持有业务真值 |
| 禁止 | 用聊天内容替代合同、资金、支付、预授权、最终金额、保证金或发票真值 |

## 4. 当前最小闭环

当前最小闭环只覆盖：

1. 竞标方申请参与。
2. 发布方审核通过。
3. 竞标方提交报价和三份竞标资料。
4. 发布方逐项预览并确认三份竞标资料。
5. 竞标方收到资料已确认和预授权提醒。
6. 竞标方进入真实预授权页面或测试授权路径。
7. Server 确认预授权满足后，项目级自由发送开启。
8. 消息楼展示状态、入口、未读和跳转，但不持有资金真值。

## 5. 资料确认交互裁决

三份竞标资料为：

1. 项目理解。
2. 报价表。
3. 进度安排。

发布方必须逐项进入资料确认页，App 内预览资料后点击“确认无误”，Server 返回确认成功后该项才能变为已确认状态。

资料确认 UI 状态：

| 状态 | 颜色语义 | 是否可发送自由聊天 | 真值来源 |
|---|---|---|---|
| 待确认 | 黄色 | 否 | Server material review pending |
| 已确认 | 绿色 | 仍需看预授权状态 | Server material review confirmed |
| 需补充 | 风险色 | 否 | Server material review needs_supplement |
| 暂不可读 | 弱化 | 否 | BFF/Server read failure |

Flutter 不得因用户打开资料、预览资料、点击入口或本地缓存而提前把状态染绿。

## 6. 预授权入口裁决

4000 元竞标服务费预授权是额度确认，不是扣款。

资料确认完成后，消息楼可以出现：

1. 发布方确认资料完成的业务提示。
2. 系统预授权提醒。
3. 预授权用途说明的折叠区域。
4. “去完成预授权”按钮。

按钮必须跳转到 Server/BFF 给出的真实 routeTarget。当前阶段可以接入测试授权或沙箱授权；不得用 Flutter 本地状态伪造授权完成。

## 7. 我的楼真实页面裁决

`我的楼` 至少需要承接以下真实入口：

| 页面 | 承接能力 | 当前裁决 |
|---|---|---|
| 我的项目 | 发布方项目、资料确认、后续合同入口 | P0 必须可达 |
| 我的竞标 | 竞标方提交记录、资料状态、预授权状态 | P0 必须可达 |
| 资金只读状态 | 200 元诚意金、4000 元预授权、成交确认进度只读 | P0 可折叠展示 |
| 预授权记录详情 | 创建、冻结、轮询、释放状态 | P0/P1 视测试授权裁决 |
| 合同金额确认 | finalConfirmedAmount 双方确认 | 后续专项 |
| 钱包 / 发票 / 保证金 / 结算 | 真实资金后链路 | 后续专项 |

## 8. 消息楼 No-Go

消息楼不得：

1. 持有最终合同金额。
2. 持有预授权真值。
3. 持有支付扣费真值。
4. 持有保证金、钱包、发票、结算真值。
5. 用聊天消息替代合同确认或金额确认。
6. 用 Flutter 本地状态模拟业务完成。
7. 变成完整 IM、私信、群聊或客服系统。
8. 绕过我的楼真实页面主路径。

## 9. Contracts 裁决原则

优先不新增接口。

仅当现有 contracts 无法表达以下内容时，才进入 OpenAPI 变更：

1. 新 routeTarget actionKey。
2. 新 Server-owned status / enum。
3. 新 command path。
4. 新 read model 字段。
5. 新 error code。

不得通过消息楼接口新增支付、合同、钱包、发票、保证金真值写入路径。

## 10. 测试与 UAT 门禁

进入云端 UAT 前必须满足：

1. `flutter clean && flutter pub get` 后构建本地 debug / 测试包。
2. BFF / Server 通过部署前 diff 审查。
3. tunnel 基础健康检查通过。
4. 双账号使用干净测试项目。
5. 预授权测试路径必须是 Server-owned test authorization 或支付沙箱。
6. 未授权前发送必须被 Server 拒绝，不落库、不 unread。
7. 授权后自由发送开启必须来自 Server truth。

## 11. 四类判断

| 判断项 | 裁决 |
|---|---|
| 最稳 | 先冻结入口地图，再按 P0/P1/P2 拆真实页面与消息楼跳转 |
| 最低成本 | 只修 P0：资料确认页、预授权提醒、我的竞标入口、资金说明折叠 |
| 最适合当前阶段 | P0 最小闭环 + 我的楼真实入口骨架 |
| 风险最大 | 一次性把支付、钱包、保证金、发票、最终金额和 IM 都塞进消息楼 |
