# 编辑项目预发布信息补充与报价依据资料附件内嵌 UI 冻结 Addendum

## 1. 本轮最小闭环

本轮只处理 Flutter 前端展示层：

- 页面对象：编辑项目页中“草稿 -> 预发布”承接页面。
- 展示名称：`编辑项目 草稿 -> 预发布信息补充页`。
- 主按钮文案：`确认保存并进入预发布信息补充页`。
- 草稿编辑页：只调整路径和按钮文案，基础信息、项目地点与范围、计划时间、补充说明继续保留原可编辑表单。
- 预发布信息补充页：基础信息、项目地点与范围、计划时间、补充说明不再第三次完整铺开为大表单，改为折叠只读信息框。
- 报价依据资料 checklist：待上传附件内嵌到当前选中的资料类型项下，不再独立展示“待上传附件（N）”大模块。

## 2. 不做事项

本轮不做以下事项：

- 不修改 BFF。
- 不修改 Server。
- 不修改 OpenAPI / contracts。
- 不修改数据库。
- 不修改项目状态机。
- 不修改保存草稿、保存到预发布、正式发布的真实 action。
- 不修改上传三步流：init -> direct upload -> confirm。
- 不修改 FileAsset / Evidence 真值。
- 不新增资料类型。
- 不新增假入口、假步骤、假发布能力。
- 不限制现有允许上传的常见文件格式。

## 3. 页面结构冻结

编辑项目页当前承接逻辑保持现有真实路由和真实 action。

展示层调整如下：

- 顶部路径统一为：`编辑项目 草稿 -> 预发布信息补充页`。
- 底部主按钮统一为：`确认保存并进入预发布信息补充页`。
- 次按钮保留：`仅保存草稿`。
- 低频入口保留：`查看我的项目详情`。
- 草稿编辑页中部继续保留原可编辑表单，不得改成只读。
- 预发布信息补充页中部不再完整铺开展会、品牌、项目类型、预算、面积、地点、计划时间、补充说明等编辑表单。
- 预发布信息补充页中部改为折叠只读信息框：
  - 折叠态展示关键摘要。
  - 展开态展示只读详情。
  - 若需要修改，展示返回现有真实编辑入口；不得把草稿编辑页的展开内容改成只读。

## 4. 字段真源

本轮只使用当前 Flutter 页面已经回填的真实字段：

- 展会名称。
- 品牌名称。
- 项目类型。
- 预算金额。
- 项目面积。
- 省 / 市 / 区县。
- 详细地址。
- 范围说明。
- 计划开始日期。
- 计划结束日期。
- 详细时间。
- 补充说明。

展示层不得新增 savedAt、updatedAt 或任何新的业务字段。

## 5. 按钮能力冻结

以下按钮只复用现有真实 action：

- `确认保存并进入预发布信息补充页`：继续调用现有保存到预发布的提交 action。
- `仅保存草稿`：继续调用现有保存草稿 action。
- `查看我的项目详情`：继续调用现有我的项目详情路由。
- `返回草稿编辑`：仅在预发布信息补充页的只读核对场景使用，继续指向现有项目编辑路由，不造新路由。

按钮文案调整不得改变真实状态机或提交接口。

## 6. 报价依据资料附件规则

报价依据资料 checklist 保留五类资料：

- 效果图。
- 尺寸图 / 施工图。
- 材质图 / 材料样板。
- 设备物料清单。
- 服务清单。

附件归属冻结：

- 待上传附件属于当前选中的资料类型。
- 当前选中项显示待上传数量，例如：`待上传 1`。
- 当前选中项显示选择状态：`已选择`。
- 未选择或无待上传附件的资料类型显示：`待补充 / 添加`。
- 上传完成后的正式资料列表仍以后端正式列表回读为准。

文件格式冻结：

- 常见文件格式沿用现有上传能力。
- 前端不得因为资料类型名称和文件格式不完全一致而阻断上传。
- XLSX、PDF、图片、文档、压缩包等常见资料文件均按现有能力处理。

附件操作冻结：

- `预览` 必须保留，作为优先查看能力。
- `打开` 必须保留，作为系统兜底能力。
- `移除` 必须保留，视觉可弱化但不能减少能力。
- `上传并形成正式附件` 必须保留，且作为主按钮独占一行。
- `继续添加附件` 必须保留。
- `更换资料类型` 必须保留。
- `刷新正式列表` 必须保留。

## 7. 验收标准

本轮验收必须满足：

- 只改 Flutter 展示层和本 addendum。
- 不改 BFF / Server / OpenAPI / 数据库。
- 不改上传三步流和 FileAsset / Evidence 真值。
- 草稿编辑页仍可直接编辑基础信息、项目地点、计划时间和补充说明。
- 预发布信息补充页不再第三次完整铺开基础信息大表单。
- “待上传附件（N）”独立大模块消失。
- 待上传附件显示在当前资料类型项内部。
- 五类资料完整保留。
- `预览 / 打开 / 移除` 可见且能力不减少。
- 主按钮 `上传并形成正式附件` 独占一行。
- bottom nav 不遮挡操作区。

## 8. 风险与降级

- 如果预发布信息补充页需要回到编辑，降级为现有真实编辑入口，不新增假入口。
- 如果某类文件无法应用内预览，保留“打开”作为系统兜底。
- 如果附件组件被其他页面复用，本轮只做局部展示分支，避免影响竞标提交页或我的项目详情页。
- 如果云端未部署或隧道不可用，本轮不能宣称云端联调通过，只能记录云端状态未验证。
