# 《七角色组织冻结单》

## 1. 固定角色

| 角色 | 主要职责 | 禁止事项 |
|---|---|---|
| 总控 | 统一当前有效真源，冻结主线与边界，输出母文件/总表/派工单，组织结果校验，决定是否进入联调发布 | 不长期替代执行角色施工；不把 docs-only freeze 写成 unlock；不忘记 `我的楼` 是当前唯一主线 |
| 总控文书冻结 | 收紧真源引用链，修订母文件、边界冻结单、closure、索引、阅读顺序、真源挂图 | 不独立加 scope；不独立发 implementation unlock；不替代总控做最终 Go / No-Go |
| 前端 Agent（仅本地） | 本地 Flutter 页面、组件、路由、交互、消费层与隧道联调验证 | 不在本地写 BFF / 后端；不自创业务规则；不误开放隐藏 building |
| 后端 Agent（仅云端） | 云端领域模型、数据库、状态机、权限、审计、核心接口 | 不把关键状态下沉到前端或 BFF；不新造 second truth；不本地施工后冒充云端成果 |
| BFF Agent（仅云端） | 云端聚合接口、整形、错误归一、上下文拼装、上传签名 | 不持有业务真相；不维护第二状态机；不把 admin review 路由到 BFF |
| 结果校验 Agent | 独立复核重复施工、越权施工、scope 漂移、formal surface 误判、入口 owner / truth owner 混写 | 不替执行角色圆场；不只看回执文字；不跳过主线偏离检查 |
| 联调发布 Agent | 基于本地前端 + 云端 BFF/后端真实拓扑做联调、证据收集、回滚核验 | 不把“代码已写”当“可上线”；不在结果校验通过前抢先进入发布口径 |

## 2. 总控与总控文书冻结的区别

| 项目 | 总控 | 总控文书冻结 |
|---|---|---|
| 是否决定当前主线 | 是 | 否 |
| 是否决定 Go / No-Go | 是 | 否 |
| 是否可发 implementation unlock | 仅总控可裁决 | 不可 |
| 是否可写业务实现代码 | 原则上否，不长期下沉施工 | 否 |
| 是否负责真源整编、索引、阅读顺序 | 负责裁决与最终收口要求 | 负责正式整编与文书收紧 |
| 是否可扩 scope | 否，必须受当前主线约束 | 否，且不得自行加 scope |

## 3. 派工链

1. 总控先输出：
   - 接管声明
   - 七角色组织冻结单
   - 基线裁决单
   - 母文件
   - 总表
   - 三栏裁决表
   - Round 1 增量派工单
2. 总控把文书收口任务派给：
   - `总控文书冻结`
3. 总控在文书收口完成且阶段门禁通过后，再分别派给：
   - `前端 Agent`
   - `后端 Agent`
   - `BFF Agent`
4. 执行角色回执后，总控把复核任务派给：
   - `结果校验 Agent`
5. 只有总控明确放行且结果校验通过后，才派给：
   - `联调发布 Agent`

## 4. 回执链

- `总控文书冻结 -> 总控`
  - 只回正式版文书包、引用链修订清单、阅读顺序、索引与挂图
- `前端 Agent -> 总控`
  - 回前端变更清单、本地验证结果、隧道联调准备状态
- `后端 Agent -> 总控`
  - 回云端后端变更清单、接口/查询/状态真相对齐结果、运行态证据
- `BFF Agent -> 总控`
  - 回聚合路径、错误归一、app-facing 整形结果与联调证据
- `结果校验 Agent -> 总控`
  - 只回：
    - `通过`
    - `有条件通过`
    - `不通过`
- `联调发布 Agent -> 总控`
  - 回真实联调证据、环境核验结果、回滚方案与发布门禁结论

## 5. 验收链

1. 总控验收文书冻结结果：
   - 看是否收紧口径
   - 看是否新增 scope
   - 看是否改写总控裁决
2. 总控验收执行回执：
   - 看是否在允许目录内施工
   - 看是否沿用旧资产而非重复建设
   - 看是否把未来位写成当前已开放
3. 结果校验 Agent 独立复核：
   - 不受执行回执自述支配
4. 总控基于结果校验作最终阶段裁决
5. 联调发布 Agent 只在前四步完成后介入

## 6. 当前环境边界

- 前端 Agent 只在本地工作区施工：
  - `apps/mobile/**`
- 后端 Agent 与 BFF Agent 只在云端工作区施工：
  - `apps/server/**`
  - `apps/bff/**`
- 统一访问验证隧道：
  - `ssh -N -L 8080:127.0.0.1:80 root@47.108.180.198`
- 本地验证统一访问：
  - `http://127.0.0.1:8080`

## 7. 当前禁止的角色漂移

- 总控不得长期替代前端、后端、BFF、校验、发布角色施工
- 总控文书冻结不得替代总控做主线裁决
- 前端不得把 `我的项目`、`项目工作台`、公域项目展示混成同一页面语义
- 后端不得为 `我的项目` 新造 my-project-only table / snapshot / second state machine
- BFF 不得持有 certification truth / review truth / eligibility truth
- 结果校验不得无证据放行
- 联调发布不得抢跑到主线闭环之前

## 8. Formal Conclusion

- 当前正式结论如下：
  - 本项目当前固定为 `7 角色`
  - `总控` 与 `总控文书冻结` 已正式分离
  - 派工链、回执链、验收链按本文件执行
  - 任一角色越权、串岗、扩 scope，均视为流程违规
