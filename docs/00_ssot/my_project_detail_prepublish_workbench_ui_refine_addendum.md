# My Project Detail Prepublish Workbench UI Refine Addendum

状态：本轮冻结

## 1. 本轮最小闭环

本轮只优化 `我的项目详情（预发布补资料并发布页）` 的 Flutter 展示层，把当前页面从多模块纵向铺开收敛为 `预发布补资料工作台`。

最小闭环：

- 项目基础信息只展示核心摘要，其余字段默认折叠。
- 发布进度只做 5 步状态展示，不展示长说明。
- 报价依据资料作为主任务区，继续使用五类 checklist。
- 发布前待办改为紧凑列表；宽屏可双列，窄屏必须单列。
- 公共资源下载区默认折叠，只展示真实分类数量和展开入口。

本轮只改 Flutter 展示层，不改业务真值。

## 2. 字段真源

| 展示项 | 真源 | 本轮规则 |
| --- | --- | --- |
| 项目名称 | `publicProject.title` | 只展示真实返回值，缺失显示 `未提供` |
| 项目编号 | `publicProject.projectNo` | 只展示真实返回值，缺失显示 `未提供` |
| 当前阶段 | `publicProject.state` 经前端 stage label 映射 | 不新增状态 |
| 待处理事项 | 当前阶段、报价依据资料回读、诚意金绿色通道回读组合派生 | 不造待办 |
| 报价依据资料数量 | 正式附件列表按 `attachmentKind` 统计 | 待上传草稿附件不算正式已补齐 |
| 公共资源数量 | 公共资源真实列表按 `resourceCategory` 统计 | 不固定写死数量 |
| 发布确认状态 | 当前项目阶段、三类必传资料、绿色通道表态组合派生 | 未满足时不可提交 |

## 3. 报价依据资料规则

五类资料继续保留：

- 效果图
- 尺寸图 / 施工图
- 材质图 / 材料样板
- 设备物料清单
- 服务清单

本轮前端发布确认门禁只看三类必传正式附件：

- 效果图
- 尺寸图 / 施工图
- 材质图 / 材料样板

设备物料清单、服务清单是建议补充项，入口和能力保留，不作为本轮硬门槛。

## 4. 组件边界

### 项目基础信息卡

保留：

- 项目名称
- 项目编号
- 当前阶段
- 待处理事项
- 展开全部信息
- 补充报价依据资料

其他字段只进入折叠详情，不删除字段。

### 发布进度

保留 5 步：

- 基础信息
- 报价依据资料
- 诚意金
- 确认发布
- 已发布

预发布补资料页当前步骤高亮为 `报价依据资料`，不展示长说明卡。

### 发布前待办

保留真实待办：

- 报价依据资料：显示 `0/3 必传已补齐` 等真实派生状态。
- 项目真实性诚意金绿色通道：显示云端回读状态。
- 绿色通道表态：显示支持 / 暂不支持真实回读数量。
- 发布确认：未满足时按钮不可提交。

### 公共资源下载区

默认折叠。

只展示真实分类数量：

- 合同模板 N
- 流程图与说明 N
- 公共资料 N

展开后再显示真实列表和下载入口，不伪造下载链接。

## 5. 不做事项

本轮不做：

- 不改 BFF。
- 不改 Server。
- 不改 OpenAPI / contracts。
- 不改数据库。
- 不改项目状态机。
- 不改诚意金规则。
- 不改上传三步流。
- 不新增假状态、假资料、假发布能力。
- 不新增公共资源链接。
- 不新增 `更换资料` 路由；如无真实能力，不强行展示。
- 不删除上传、预览、打开、移除、刷新、发布、撤回、作废归档、公共资源下载能力。

## 6. 验收标准

- 页面长度明显短于改造前。
- 报价依据资料成为主任务区。
- 发布进度无长说明。
- 公共资源默认折叠。
- 发布确认未满足时不可提交。
- 窄屏下发布前待办不拥挤，必须单列。
- bottom nav 不遮挡最后一项。
- `flutter analyze --no-pub` 无本轮新增问题。
- 相关 Flutter tests 通过，或明确记录既有环境阻塞。
