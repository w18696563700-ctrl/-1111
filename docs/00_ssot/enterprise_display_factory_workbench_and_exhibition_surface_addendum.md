# 企业展示优秀工厂工作台与展览页当前真值补充记录

## 目的
- 冻结 `优秀工厂工作台` 与 `展览页优秀工厂列表/详情` 的当前真值边界。
- 明确工厂图片区块、列表/详情回显、以及测试期自动审核通过后的上线行为。
- 防止后续改动再次误伤工厂板块。

## 范围
- `apps/mobile` 工厂工作台、工厂列表、工厂详情消费面
- `apps/bff` 企业展示工厂读写聚合层
- `apps/server` 企业展示工厂真值、自动审核、发布可见性

## 当前真值规则

### 1. 工厂图片区块的真值
- 工厂图片区块承载的是 `showcaseImageFileAssetIds`，不是 `企业画册 albumImageFileAssetIds`。
- 工厂图片区块回读时，后端必须同时返回：
  - `showcaseImageFileAssetIds`
  - `showcaseImageUrlMap`
- `Flutter App` 回显优先级必须是：
  - 本地 `bytes`
  - 远端 `imageUrl`
  - 占位图
- 只返回 `fileAssetId` 不返回可展示 URL，视为合同不完整。

### 2. 工厂图片区块必须可独立保存
- 工厂图片区块的“确认上传”只负责保存 `showcaseImageFileAssetIds`。
- 不允许把工厂图片区块上传确认与整份工厂画像必填校验绑定。
- `processTypes`、`coreProducts` 是工厂画像提交边界，不是工厂图片区块确认边界。
- 任何把“确认上传工厂图片”重新绑回整份工厂画像校验的改动，都视为误伤。

### 3. 工厂首次提交自动审核通过后的可测要求
- 当工厂展示首次提交满足自动审核通过条件时：
  - 申请状态允许直接进入 `approved`
  - 列表展示档必须同步进入 `published + visible`
- 只做 `approved` 而不让 `listing` 进入 `published + visible`，视为测试链未闭合。
- 本条用于保障 `展览页优秀工厂列表/详情` 可以直接承接自动审核后的测试结果。

### 4. 展览页工厂列表与详情只消费正式公开真值
- `优秀工厂列表` 与 `工厂详情` 只消费正式公开真值。
- 不允许把未发布草稿、未应用的 current change、或私有 workbench 草稿直接混入公开列表/详情。
- 公开页的工厂图片必须消费正式可读 URL，不允许退回裸 `fileAssetId` 或私有桶直链。

## 保护线
- 不得删除 `showcaseImageUrlMap` 回读链。
- 不得把工厂图片区块重新合并为“保存整份画像”。
- 不得把“自动审核通过后可测试工厂列表/详情”的链路重新拆断。
- 后续若要更改上述规则，必须先更新本记录，再改代码。
