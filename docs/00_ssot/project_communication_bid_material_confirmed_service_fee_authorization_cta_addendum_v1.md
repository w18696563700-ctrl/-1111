# 三份竞标资料确认后预授权提醒与聊天框 CTA Addendum V1

状态：Frozen

冻结日期：2026-05-08

## 0. 总裁决

发布方将竞标方提交的三份竞标资料全部确认为 `confirmed` 后，Server 必须向竞标方所属组织写入项目沟通系统提醒。该提醒必须在竞标方即时聊天框中可见，并提供进入 `4000 元竞标服务费预授权额度` 处理入口的明确 CTA。

本 Addendum 只冻结“资料确认完成后的提醒与入口承接”，不冻结真实支付宝 / 微信预授权通道，不冻结支付扣款、服务费结算、保证金、钱包、发票、最终合同金额确认或 BidAward。

## 1. 触发条件

系统提醒的触发条件为：

1. 当前项目已存在有效 Bid。
2. 三份竞标资料均已由 Server 保存并绑定到该 Bid：
   - 项目理解。
   - 报价表。
   - 进度安排。
3. 发布方分别对三份竞标资料完成 `confirm`。
4. 三份资料在 Server read model 中均为 `confirmed`。
5. 当前竞标方尚未完成 `4000 元竞标服务费预授权额度`，项目级自由发送仍未开启。

若任一资料为 `needs_supplement`，不得触发预授权提醒；必须先走资料补充与重新确认。

## 2. 系统提醒与 CTA

系统提醒必须满足：

1. 写入 Server project communication message。
2. 生成竞标方 unread / notification，具体投影由既有消息楼 read model 承接。
3. 在竞标方即时聊天框展示为系统业务提醒，而不是普通自由聊天。
4. 展示文案必须明确：
   - 发布方已确认三份资料。
   - 下一步是完成 `4000 元竞标服务费预授权额度`。
   - 完成后项目级自由发送开启。
   - 预授权不是扣款。
5. CTA 文案固定为：`去完成预授权`。
6. CTA 必须跳转到竞标方预授权处理入口；入口不可用时必须提示原因，不得点击无反应。

## 3. 推荐文案

主提醒文案：

`发布方已确认完你的资料：项目理解、报价表、进度安排。`

补充提醒文案：

`把握好商机，请尽快完成 4000 元竞标服务费预授权额度；完成后项目级自由发送将开启。预授权不是扣款。`

若需要展示资金用途说明，应作为折叠说明或次级说明展示，不得占据聊天主操作区。

## 4. Server / BFF / Flutter 边界

### 4.1 Server

Server 是以下真值 owner：

1. 三份竞标资料确认状态。
2. 是否触发预授权提醒。
3. 系统提醒消息、unread、notification。
4. 预授权状态。
5. project communication `canSendMessage`。

Server 写入系统提醒时必须具备幂等保护，避免重复确认或重复刷新产生重复提醒。

### 4.2 BFF

BFF 只做：

1. 透出 Server 返回的系统消息 payload。
2. 透出 `requiredNextAction=complete_service_fee_authorization`。
3. 透出可用的 routeTarget 或业务入口 projection。
4. 错误归一化。

BFF 不得创造资料确认状态、预授权状态、聊天解锁状态或支付真值。

### 4.3 Flutter

Flutter 只做：

1. 渲染系统提醒。
2. 渲染 `去完成预授权` CTA。
3. 点击 CTA 后按 Server / BFF projection 跳转。
4. 确认资料成功后刷新 workbench、thread/chatAvailability 和 messages。
5. 预授权未完成前保持聊天锁定提示。

Flutter 不得本地模拟预授权完成，不得本地模拟消息已发送，不得用本地状态放开项目级自由发送。

## 5. No-Go 范围

本 Addendum 不允许扩展：

1. 真实支付宝 / 微信预授权通道。
2. 自动解冻定时规则。
3. 支付扣款。
4. 服务费结算。
5. 保证金。
6. 钱包。
7. 发票。
8. 最终合同金额确认。
9. BidAward / Order / Contract seed 语义。
10. 通用 IM / 私信 / 群聊 / 客服系统。
