# 申请参与竞标 Phase 1 L0 规则冻结单

## 0. 总裁决

- 当前是否允许保留独立「申请查看项目名称」用户入口：No-Go
- 当前是否允许进入「申请参与竞标」一期实现：Go
- 当前是否允许把资料授权、NDA、水印、下载审计扩展混入一期：No-Go
- 当前唯一业务真相 owner：Server

## 1. 当前最小闭环

一期把「申请查看项目名称」统一收敛为「申请参与竞标」：

1. 竞标方在公域项目详情点击「立即申请参与竞标」。
2. Server 创建 `BidParticipationRequest`，状态为 `pending`。
3. 发布方在消息楼 / 项目沟通中看到申请记录。
4. 发布方可查看申请方主体摘要并选择通过或拒绝。
5. 通过后自动解锁：
   - 项目真实名称；
   - 报价依据资料列表；
   - 报价依据资料文件访问；
   - 竞标提交入口。
6. 拒绝后 fail closed：
   - 项目名称仍保持脱敏；
   - 报价依据资料不可读；
   - 竞标提交不可进入。
7. 竞标方在消息楼可看到 `pending / approved / rejected` 状态；`approved` 消息 CTA 可直达竞标提交页。

## 2. 本轮做什么

| 项 | 冻结结论 |
|---|---|
| 申请主体 | 当前竞标方组织 |
| 审批主体 | 项目发布方组织 |
| 权限粒度 | `projectId + requesterOrganizationId` |
| 状态 | `pending / approved / rejected` |
| 通过后权限 | 解锁名称、报价依据资料、文件访问、竞标提交 |
| 拒绝后权限 | 保持不可见，不允许提交竞标 |
| 消息楼 | 承接申请、审批结果和 approved 后跳转 |
| 旧名称申请入口 | 用户可见入口下线；旧真值仅兼容历史数据 |

## 3. 本轮不做什么

| 不做项 | 原因 |
|---|---|
| NDA 签署 | 需要独立法律/合规模块冻结 |
| 水印与防泄漏 | 属于资料安全二期 |
| 单文件二次授权 | 一期只做项目级参与申请 |
| 支付/预授权/扣款 | 与参与申请无关，避免污染资金链 |
| 会员权益联动 | 已有独立 P0-Pay 包，不混线 |
| 完整 IM 仲裁 | 消息楼只承接状态卡和 CTA |

## 4. Server / BFF / Flutter 边界

| 层 | 边界 |
|---|---|
| Server | 唯一拥有申请状态、审批、名称解锁、资料访问、竞标提交准入真相 |
| BFF | 只转发 Server，做字段整形和中文错误，不判断是否通过 |
| Flutter | 只展示状态、发起申请、展示 CTA，不本地判定权限真相 |

## 5. 需要保留但暂不开通

- 历史 `project_name_access_requests` 数据继续兼容读取；
- 旧 `project_name_access_thread` 路由暂不物理删除，避免历史消息断链；
- 独立资料 NDA / 水印 / 下载审计保留扩展位。

## 6. 后续扩展位

- 资料水印；
- NDA 签署；
- 文件级查看审计；
- 发布方批量审批；
- 黑名单 / 信用分影响申请可见性。

## 7. 阶段门禁

| 门禁 | 结论 |
|---|---|
| L0 规则冻结 | Pass |
| L2 Contracts | Allow |
| Server 实现 | Blocked until L2/L3 passed |
| 云端写入 | Blocked until Day8 单独授权 |
