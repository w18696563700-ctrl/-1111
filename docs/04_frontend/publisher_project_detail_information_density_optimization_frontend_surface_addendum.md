# 《发布方项目详情页信息密度优化 frontend surface》

## 1. 页面顺序

发布方 `我的项目详情` 页按以下顺序展示：

1. 页面导航与标题。
2. `已保存的项目基础信息摘要` 折叠卡。
3. `报价依据资料` 主任务区。
4. `公共资源下载区` 弱化展示。
5. 其他既有底部导航或系统级区域。

本页不再展示大面积 `项目沟通` 主入口卡片。

## 2. 基础信息折叠状态

`已保存的项目基础信息摘要` 默认折叠。

折叠态至少保留：

- 项目名称
- 项目编号
- 当前状态
- 关键摘要，优先使用预算、面积、地点或时间中已有字段
- 展开 / 收起按钮

展开态保留现有基础信息字段，不删除项目核对能力。

如已有 `补充报价依据资料` 或同等 handoff 按钮，可继续保留在该卡片底部，但不得成为页面唯一入口；报价依据资料区本身仍必须在下方直接可见。

## 3. 报价依据资料主任务区

标题固定为：

- `报价依据资料`

定位文案应短，不重复解释完整上传流程。推荐文案：

> 请补充效果图、尺寸图 / 施工图、材质图 / 材料样板、设备物料清单、服务清单，作为接单方报价依据。

资料类型入口固定为 5 类：

- 效果图
- 尺寸图 / 施工图
- 材质图 / 材料样板
- 设备物料清单
- 服务清单

Flutter 不得展示或写入 `other_material` 作为新主路径。

## 4. 资料卡片字段

正式资料列表中的每张卡片改为全宽卡片。

每张卡片字段顺序固定为：

1. 资料名称
2. 文件名
3. 文件类型
4. 可见范围
5. 排序序号
6. 创建时间
7. 操作按钮

`资料名称` 从 `attachmentKind` 派生，不新增接口字段，不要求用户填写标题。

映射固定为：

| attachmentKind | 资料名称 |
| --- | --- |
| `effect_image` | 效果图 |
| `construction_doc` | 尺寸图 / 施工图 |
| `material_sample` | 材质图 / 材料样板 |
| `equipment_material_list` | 设备物料清单 |
| `service_list` | 服务清单 |

`可见范围` 文案固定使用已有权限真相表达，推荐：

- `仅 owner 私域可见；接单方在竞标第二步查看`

若 Flutter 当前只能拿到 owner-private 字段，则可显示：

- `仅 owner 私域可见`

不得在 Flutter 中伪造新的可见范围状态。

## 5. 按钮保留

报价依据资料区保留既有 owner 操作能力：

- 选择资料类型
- 选择文件
- 上传 / 确认上传
- 刷新正式列表
- 预览图片 / 预览文书
- 删除当前资料

按钮文案可按资料类型自适配：

- 图片类使用 `预览图片`
- 文书 / 表格类使用 `预览文书`

不得把 owner 管理按钮泄漏到接单方竞标页。

## 6. 项目沟通当前页处理

发布方项目详情页隐藏 `项目沟通` 主入口卡片。

不得删除以下能力或代码路径：

- 消息 building 中的项目沟通入口
- 项目沟通 thread
- 项目沟通消息列表
- 项目沟通后续轻量跳转扩展位

本轮不新增 Flutter 聊天能力、不改变 routeTarget、不改 BFF / Server。

## 7. 公共资源下载区弱化

`公共资源下载区` 保留，但不作为本页主任务。

需要裁剪的内容：

- 删除或隐藏大段说明文案。
- 删除或隐藏对共享资料规则的重复解释。
- 不再与报价依据资料争夺页面第一屏。

需要保留的内容：

- 资源分类 chips。
- 共享资料列表。
- 下载按钮。
- 必要的空态和错误态。

推荐短文案：

> 可下载平台共享模板与公共资料。

## 8. 空态与错误文案

报价依据资料空态推荐：

- `当前项目暂未上传报价依据资料。`

公共资源空态推荐：

- `暂无可下载公共资源。`

项目沟通不在本页出现空态；沟通入口以消息 building 为准。

## 9. 测试验收点

Flutter 目标测试应覆盖：

- 基础信息摘要默认折叠。
- 页面不展示大面积 `项目沟通` 主入口卡片。
- `报价依据资料` 位于 `公共资源下载区` 之前。
- 资料入口包含 5 类，且不包含 `other_material` 用户文案。
- 正式资料卡片为全宽展示。
- 正式资料卡片展示 `资料名称` 与文件名。
- 预览与删除按钮仍存在。
- 公共资源长说明文案不再出现。
- 接单方竞标页不获得 owner 管理按钮。

## 10. 不做项

本 L5 明确不授权：

- BFF 改动。
- Server 改动。
- 数据库改动。
- OSS 改动。
- 新增资料标题字段。
- 自定义资料标题输入框。
- 删除项目沟通能力。
- 把公共资源合并进报价依据资料。

## 11. 执行结论

Flutter 可按本 surface 进入实现。

实现应优先复用既有组件、既有上传链路、既有附件 mapper 和既有 file-access 预览动作，只调整发布方项目详情页的信息层级、折叠状态、卡片布局和文案密度。
