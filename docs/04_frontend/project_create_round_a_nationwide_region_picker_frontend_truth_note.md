## 项目创建第一页地区选择全国省市区对齐说明

日期：`2026-04-16`

### 背景

- 展览楼创建项目第一页当前已接入全国省市目录，但 `区/县` 仍依赖少量样例城市硬编码。
- 这种实现会让页面看起来像“全国三级联动”，实际只有少量城市能补充区县，不符合当前产品要求。

### 当前前端真值

- `项目地点与范围` 的地区选择必须承接全国行政区划数据。
- `省 / 市` 必须来自全国标准省市目录。
- `区/县` 必须跟随所选城市回填全国标准区县目录；不能继续只支持样例城市。
- `districtCode / districtName` 仍保持既有 contract：可同时为空；若提供则必须成对提交。

### UI 口径

- 地区选择入口文案必须与真实能力一致。
- 如果入口同时承担省市联动选择，不得继续只写 `选择城市` 误导用户。
- 创建页应明确表现为：
  - 先选 `省 / 市`
  - 再按当前城市补充 `区/县`

### Anti-revert Rule

- 不得把创建页地区选择退回为“全国省市 + 8 个样例区县”的伪三级联动。
- 不得删除全国区县数据承接后又改回静态样例表。
- 不得用提示文案掩盖数据范围不足的问题；真实能力不足时必须修数据源，而不是改说法糊弄过去。
